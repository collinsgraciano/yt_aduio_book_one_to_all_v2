# 数据库备份恢复迁移指南

## 1. 数据库备份

### 1.1 自动备份

```bash
# 备份到默认目录 ./backups/
bash scripts/db_backup.sh

# 备份到指定目录
bash scripts/db_backup.sh /path/to/backupdir

# 备份到指定文件
bash scripts/db_backup.sh /path/to/backup.sql.gz
```

备份脚本通过 `docker exec` 进入 `audiobook_postgres` 容器执行 `pg_dump`，导出全库数据并压缩为 `.sql.gz` 文件。自动保留最近 10 个备份。

### 1.2 手动备份

```bash
# 全库备份（压缩）
docker exec audiobook_postgres pg_dump -U audiobook_app -d audiobook \
    --no-owner --no-privileges | gzip > backup_$(date +%Y%m%d).sql.gz

# 仅备份 books 表
docker exec audiobook_postgres pg_dump -U audiobook_app -d audiobook \
    --data-only --table=public.books | gzip > books_backup.sql.gz

# 仅备份表结构
docker exec audiobook_postgres pg_dump -U audiobook_app -d audiobook \
    --schema-only --no-owner > schema.sql
```

### 1.3 定时备份（crontab）

```bash
# 编辑 crontab
crontab -e

# 每天凌晨 3 点自动备份
0 3 * * * cd /root/audiobook && bash scripts/db_backup.sh >> backups/cron.log 2>&1

# 每周日凌晨 3 点完整备份 + 清理 30 天前的备份
0 3 * * 0 cd /root/audiobook && bash scripts/db_backup.sh && find backups/ -name "*.sql.gz" -mtime +30 -delete
```

## 2. 数据库恢复

### 2.1 从备份恢复

```bash
# 恢复（会覆盖当前数据，需输入 yes 确认）
bash scripts/db_restore.sh backups/audiobook_backup_20260720_120000.sql.gz

# 恢复后重启 Web 服务
docker-compose restart web
```

恢复脚本会：
1. 断开所有数据库连接
2. 删除并重建 `audiobook` 数据库
3. 从备份文件导入数据
4. 验证表行数

### 2.2 手动恢复

```bash
# 解压并恢复
gunzip -c backup.sql.gz | docker exec -i audiobook_postgres psql -U audiobook_app -d audiobook

# 恢复单表
docker exec audiobook_postgres psql -U audiobook_app -d audiobook \
    -c "TRUNCATE public.books CASCADE;"
gunzip -c books_backup.sql.gz | docker exec -i audiobook_postgres psql -U audiobook_app -d audiobook
```

## 3. 从旧项目迁移数据

### 3.1 迁移场景

如果你之前运行的是旧版项目（`audiobook_pipeline` 或早期版本），需要将旧数据库中的数据迁移到新系统。

### 3.2 核心表说明

| 表名 | 说明 | 迁移方式 |
|------|------|----------|
| `books` | 书籍库（含 book_data JSON） | 直接迁移 |
| `book_processing_states` | 断点续跑状态 | 直接迁移 |
| `youtube_credentials` | YouTube OAuth Token | 直接迁移 |
| `modelscope_tokens` | AI 生图 Token | 直接迁移 |
| `channel_runtime_settings` | 频道运行时配置 | 直接迁移 |
| `task_queue` | 原始任务队列 | 直接迁移 |
| `audiobook_chapters` | TG 缓存章节 | 特殊处理（见 3.4） |
| `channels` | 频道注册 | 重新创建（需上传 OAuth client_secret） |
| `channel_configs` | 频道运行配置 | 重新创建（通过 Web 界面配置） |
| `global_settings` | 全局共享设置 | 重新配置（通过 Web 界面） |

### 3.3 使用迁移脚本

```bash
# 设置旧数据库连接信息
export OLD_PG_HOST=127.0.0.1        # 旧数据库地址
export OLD_PG_PORT=5432              # 旧数据库端口
export OLD_PG_USER=postgres          # 旧数据库用户
export OLD_PG_DB=audiobook           # 旧数据库名
export OLD_PG_PASSWORD=your_old_password

# 执行迁移
bash scripts/migrate_data.sh
```

如果旧数据库在远程服务器：

```bash
export OLD_PG_HOST=85.121.48.55
export OLD_PG_PASSWORD=your_remote_password
bash scripts/migrate_data.sh
```

迁移脚本执行流程：
1. 测试旧数据库连接
2. 逐表导出（`pg_dump --data-only --column-inserts`）
3. 验证新数据库表结构
4. 清空目标表后导入
5. 逐表对比行数验证

### 3.4 迁移 audiobook_chapters（TG 缓存）

`audiobook_chapters` 表存储了 Telegram 已降噪音频的 `file_id`，是 TG 缓存下载的关键数据。如果旧项目已上传大量章节到 TG，需要单独迁移：

```bash
# 1. 在旧数据库导出
PGPASSWORD=your_old_password pg_dump -h OLD_HOST -U OLD_USER -d OLD_DB \
    --data-only --column-inserts --no-owner --no-privileges \
    --table=public.audiobook_chapters > chapters_export.sql

# 2. 导入到新数据库
docker exec -i audiobook_postgres psql -U audiobook_app -d audiobook \
    -c "TRUNCATE public.audiobook_chapters CASCADE;"
docker exec -i audiobook_postgres psql -U audiobook_app -d audiobook < chapters_export.sql

# 3. 验证
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "
    SELECT COUNT(*) AS total,
           COUNT(telegram_file_id) AS cached,
           COUNT(*) FILTER (WHERE upload_status = 'uploaded') AS uploaded
    FROM audiobook_chapters;
"
```

### 3.5 迁移后配置

迁移完成后，以下内容需要在 Web 界面中重新配置：

1. **全局设置**（`global_settings` 表）：
   - 进入「设置」页面
   - 配置 `MODELSCOPE_TOKEN`、`TG_BOT_TOKEN` 等参数
   - 这些参数不会从旧数据库迁移（旧项目可能没有此表）

2. **频道注册**（`channels` 表）：
   - 进入「频道管理」→「添加频道」
   - 填写频道名、显示名
   - 上传 Google OAuth `client_secret.json`
   - 完成 OAuth 授权

3. **频道运行配置**（`channel_configs` 表）：
   - 进入频道详情 → 「运行配置」标签页
   - 修改 95+ 参数（下载、降噪、BGM、封面、上传等）
   - 保存配置

### 3.6 验证迁移结果

```bash
# 查看各表行数
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "
    SELECT relname AS table_name, n_live_tup AS row_count
    FROM pg_stat_user_tables
    ORDER BY n_live_tup DESC;
"

# 检查 YouTube 凭证是否可用
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "
    SELECT channel_name, updated_at FROM youtube_credentials;
"

# 检查 TG 缓存章节数
docker exec audiobook_postgres psql -U audiobook_app -d audiobook -c "
    SELECT COUNT(*) AS total,
           COUNT(telegram_file_id) AS cached
    FROM audiobook_chapters;
"

# 重启 Web 服务
docker-compose restart web
```

## 4. 数据库管理常用命令

### 4.1 连接数据库

```bash
# 进入 PostgreSQL 交互式终端
docker exec -it audiobook_postgres psql -U audiobook_app -d audiobook

# 从宿主机连接（需端口 5432 已暴露）
psql -h 127.0.0.1 -U audiobook_app -d audiobook
```

### 4.2 重置卡住的任务

```sql
-- 重置卡住的 HF 外包任务（超过 24 小时未完成）
UPDATE hf_jobs
SET status = 'pending', worker_id = NULL, claimed_at = NULL,
    retry_count = retry_count + 1
WHERE status = 'processing'
  AND claimed_at < now() - interval '24 hours';

-- 重置卡住的 audiobook_chapters（超过 24 小时未完成上传）
UPDATE audiobook_chapters
SET upload_status = 'pending', worker_id = NULL, claimed_at = NULL, error_message = NULL
WHERE upload_status = 'processing'
  AND claimed_at < now() - interval '24 hours';

-- 清理过期 OAuth state
DELETE FROM oauth_states WHERE created_at < now() - interval '10 minutes';
```

### 4.3 完全重置数据库

```bash
# 停止并删除所有容器和数据卷（会删除所有数据！）
docker-compose down -v

# 重新启动（init-db.sql 自动执行）
bash scripts/git-server-deploy.sh
```

> **警告**：`docker-compose down -v` 会删除 `postgres_data` 卷中的所有数据。执行前请先备份！

### 4.4 查看数据库大小

```sql
-- 数据库总大小
SELECT pg_size_pretty(pg_database_size('audiobook')) AS db_size;

-- 各表大小（含索引）
SELECT schemaname||'.'||relname AS table_name,
       pg_size_pretty(pg_total_relation_size(relid)) AS total_size,
       pg_size_pretty(pg_relation_size(relid)) AS table_size,
       n_live_tup AS row_count
FROM pg_stat_user_tables
ORDER BY pg_total_relation_size(relid) DESC;
```
