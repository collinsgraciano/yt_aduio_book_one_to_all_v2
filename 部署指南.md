# 部署指南

## 架构概览

| 组件 | 技术 | 说明 |
|------|------|------|
| Web 服务 | FastAPI + Uvicorn | 端口 59386，Jinja2 服务端渲染 |
| 数据库 | PostgreSQL 16 | Docker 内置，端口 5432 |
| HF 中继调度器 | Flask | 端口 38080，HF 外包任务调度 + 凭证中继 |
| 任务执行 | Python `threading.Thread` | 无需 Redis/Celery |
| 容器 | Docker Compose | `web` + `postgres` + `vps-relay` 三个服务 |

```
用户浏览器 → :59386 → FastAPI (web 容器)
                        ├── Jinja2 页面渲染
                        ├── API 路由 (频道/任务/书籍/配置/测试)
                        ├── 后台线程 (Pipeline 执行)
                        └── PostgreSQL (postgres 容器)
                              ├── books / audiobook_chapters
                              ├── channels / channel_configs
                              ├── run_tasks / run_task_logs
                              ├── global_settings / oauth_states
                              └── hf_jobs / hf_worker_stats

HF Space (远程) → :38080 → VPS 中继 (vps-relay 容器)
                               ├── 任务调度（派发 HF 外包任务）
                               ├── TG API 中继（凭证不落地）
                               ├── YouTube OAuth 中继（短期 token 分发）
                               └── 配置/密钥分发
```

## 首次部署

### 前置条件

- VPS 或本地机器已安装 Docker + Docker Compose
- 已有 Git 仓库（GitHub/Gitea 等）

### 步骤 1：服务器克隆仓库

```bash
git clone https://github.com/YOUR_USER/YOUR_REPO.git /root/audiobook
cd /root/audiobook
```

### 步骤 2：配置环境变量

```bash
cp .env.example .env
nano .env
```

需要修改的配置：

| 变量 | 说明 | 示例 |
|------|------|------|
| `POSTGRES_PASSWORD` | 数据库密码（首次启动创建，之后不可改） | `my_strong_password_123` |
| `SECRET_KEY` | 加密密钥（64 位随机字符串） | `openssl rand -hex 32` 生成 |
| `BASE_URL` | 外部访问地址（OAuth 回调用） | `http://123.45.67.89:59386` |
| `APP_PASSWORD` | Web 界面登录密码（可选，默认 `inriynisse`） | `mypassword` |
| `WORKER_URLS` | HF 统一 Worker 地址（可选，部署 HF Space 后填） | `https://user-audiobook-worker-1.hf.space` |
| `RELAY_WEB_PASSWORD` | 中继管理面板密码（可选，留空=无密码） | `relay_admin_123` |

> **重要**：`BASE_URL` 必须与 Google Cloud Console 中 OAuth 重定向 URI 一致。
> 回调地址格式为 `{BASE_URL}/api/oauth/callback`。

### 步骤 3：一键部署

```bash
bash scripts/git-server-deploy.sh
```

脚本自动完成：
1. `git pull` 拉取最新代码（脚本自身更新会自动重新执行）
2. 检查 `.env` 配置
3. 预下载 DeepFilter 二进制 + BGM 音乐库（持久化到宿主机 `data/` 目录）
4. 智能构建 Docker 镜像（仅在依赖/源码变更时重建）
5. 启动 `web` + `postgres` + `vps-relay` 容器
6. 健康检查（等待 Web 服务和中继调度器就绪）

首次部署约需 5-10 分钟（下载镜像 + 构建依赖 + 下载 DeepFilter/BGM）。
后续日常更新通常只需 10-30 秒（跳过构建，仅重启）。

### 步骤 4：验证

```bash
# 查看服务状态（应显示 3 个容器）
docker-compose ps

# 查看日志
docker-compose logs -f web
docker-compose logs -f vps-relay

# 浏览器访问
http://YOUR_SERVER_IP:59386     # Web 管理系统
http://YOUR_SERVER_IP:38080     # HF 中继管理面板
```

首次访问需输入登录密码（默认 `inriynisse`，可在 `.env` 中修改 `APP_PASSWORD`）。

API 文档：`http://YOUR_SERVER_IP:59386/api/docs`

HF 中继管理面板：`http://YOUR_SERVER_IP:38080`（可在面板「⚙️ 配置管理」中实时修改 Worker 地址、调度参数等，无需重启）

## HF 外包部署

VPS 中继调度器已随主项目一起启动，无需单独部署。只需完成以下步骤部署 HF Space：

### 第一步：在 Hugging Face 创建 Space

1. 登录 [huggingface.co](https://huggingface.co)
2. 点击右上角头像 → New Space
3. 设置：
   - **Owner**: 你的用户名
   - **Space name**: `audiobook-worker-1`（可创建多个实现并行）
   - **License**: MIT
   - **SDK**: Docker
   - **Visibility**: Public（免费版必须 Public）或 Private（付费）

### 第二步：上传代码到 HF Space

HF Space 的 Docker 构建上下文是 Space 仓库根目录，但我们的 Dockerfile 需要项目根目录作为上下文（因为要 COPY pipeline/）。

```bash
# 1. 克隆 HF Space 仓库
git clone https://huggingface.co/spaces/你的用户名/audiobook-worker-1
cd audiobook-worker-1

# 2. 复制文件
cp /path/to/project/hf_workers/unified_worker/Dockerfile .
cp /path/to/project/hf_workers/unified_worker/app.py .
cp /path/to/project/hf_workers/unified_worker/runner.py .
cp /path/to/project/hf_workers/unified_worker/requirements.txt .
cp /path/to/project/hf_workers/shared.py .
cp -r /path/to/project/pipeline/ ./pipeline/

# 3. 修改 Dockerfile 中的 COPY 路径
#    将 COPY hf_workers/unified_worker/requirements.txt 改为 COPY requirements.txt
#    将 COPY hf_workers/unified_worker/app.py 改为 COPY app.py
#    将 COPY hf_workers/unified_worker/runner.py 改为 COPY runner.py
#    将 COPY hf_workers/unified_worker/shared.py 改为 COPY shared.py
#    将 COPY pipeline/ 保持不变

# 4. 在 HF Space 的 Settings → Repository secrets 中添加环境变量：
#    - POSTGRES_DSN: postgresql://audiobook_app:你的数据库密码@你的VPS公网IP:5432/audiobook
#    - VPS_RELAY_URL: http://你的VPS公网IP:38080

# 5. 提交推送
git add .
git commit -m "Initial unified worker"
git push
```

> **注意**：HF Space 的 `POSTGRES_DSN` 中需要使用 VPS **公网 IP**（不是 Docker 内部地址），因为 HF Space 在远程运行。

### 第三步：验证 Worker

```bash
# 等待 HF Space 构建完成（约 5-10 分钟），然后：
curl https://你的用户名-audiobook-worker-1.hf.space/health
# 应返回: {"ok": true, "worker_id": "hf_xxxx", "free_slots": 1, "test_free_slots": 1, ...}
```

### 第四步：配置 Worker 地址

HF Space 部署完成后，将 Worker 地址填入 VPS 中继调度器。

**方法一：通过中继管理面板（推荐）**

1. 浏览器访问 `http://你的VPS_IP:38080`
2. 进入「⚙️ 配置管理」
3. 在 `worker_urls` 中填入 HF Space 地址
4. 保存，立即生效，无需重启

**方法二：通过 API**

```bash
curl -X POST http://你的VPS_IP:38080/api/config \
  -H "Content-Type: application/json" \
  -d '{"worker_urls": ["https://你的用户名-audiobook-worker-1.hf.space"]}'
```

**方法三：编辑 .env 后重启**

在 `.env` 中设置 `WORKER_URLS`：
```bash
WORKER_URLS=https://你的用户名-audiobook-worker-1.hf.space
```
然后重新运行 `bash scripts/git-server-deploy.sh`。

### 第五步：在后端全局设置中配置

登录 Web 管理系统 → 全局设置 → 找到 **🛰️ HF 外包** 分类：

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `VPS_RELAY_URL` | VPS 中继调度器地址 | `http://VPS_IP:38080` |
| `HF_TEST_WORKER_URLS` | HF 测试 Worker 地址（逗号分隔，与统一 Worker 相同） | `https://你的用户名-audiobook-worker-1.hf.space` |

保存后即可在前端使用 HF 外包功能。

> 详细的 HF 外包架构说明、API 参考、故障排查见 [hf_workers/README.md](hf_workers/README.md)。

## 开发模式

本地开发使用热重载模式，修改 `backend/` 下的 Python 代码后自动重载：

```bash
# 确保 .env 已配置
docker-compose -f docker-compose.yml -f docker-compose.dev.yml up
```

开发模式特点：
- `backend/` 目录挂载到容器，代码修改即时生效
- PostgreSQL 端口 5432 暴露到本地，方便用 GUI 工具调试
- Uvicorn `--reload` 自动检测文件变更

## Google OAuth 配置

### 1. 创建 Google Cloud 项目

1. 访问 [Google Cloud Console](https://console.cloud.google.com/)
2. 创建新项目（如 `audiobook-yt`）
3. 启用 YouTube Data API v3

### 2. 配置 OAuth 凭据

1. 进入「APIs & Services」→「Credentials」
2. 创建 OAuth 2.0 Client ID（Web application）
3. 添加授权重定向 URI：
   ```
   http://YOUR_SERVER_IP:59386/api/oauth/callback
   ```
   或使用域名：
   ```
   https://your-domain.com/api/oauth/callback
   ```
4. 下载 `client_secret.json`

### 3. 在系统中添加频道

1. 登录管理系统 → 频道管理 → 添加频道
2. 填写频道名和显示名
3. 上传下载的 `client_secret.json`
4. 点击「授权」→ 跳转 Google 完成授权
5. 授权成功后自动回调

> 如果 `BASE_URL` 变更，需要同步更新 Google Cloud Console 中的重定向 URI。

## 数据库管理

### 连接数据库

```bash
# 进入 PostgreSQL 容器
docker exec -it audiobook_postgres psql -U audiobook_app -d audiobook

# 从宿主机连接（需暴露端口）
psql -h 127.0.0.1 -U audiobook_app -d audiobook
```

### 数据库表说明

| 表名 | 说明 |
|------|------|
| `books` | 书籍库（含 book_data JSON、book_status 章节状态） |
| `audiobook_chapters` | TG 缓存章节（含 telegram_file_id、upload_status） |
| `channels` | 频道注册表 |
| `channel_configs` | 频道运行配置（95+ 参数 JSON） |
| `channel_runtime_settings` | 频道级运行时键值对 |
| `youtube_credentials` | YouTube OAuth Token |
| `modelscope_tokens` | AI 生图 Token |
| `book_processing_states` | 断点续跑状态 |
| `task_queue` | 原始任务队列 |
| `run_tasks` | Web 管理层任务记录（含 stop_requested） |
| `run_task_logs` | 任务日志持久化 |
| `oauth_states` | OAuth State 临时存储 |
| `global_settings` | 全局共享设置 |
| `hf_jobs` | HF 外包任务队列 |
| `hf_worker_stats` | HF Worker 统计 |

### 常用 SQL

```sql
-- 查看书籍总数
SELECT COUNT(*) FROM books;

-- 查看任务状态分布
SELECT status, COUNT(*) FROM run_tasks GROUP BY status;

-- 查看频道 OAuth 状态
SELECT channel_name, oauth_status FROM channels;

-- 重置卡住的 HF 任务
UPDATE hf_jobs SET status = 'pending', worker_id = NULL, claimed_at = NULL
  WHERE status = 'processing' AND claimed_at < now() - interval '24 hours';

-- 清理过期 OAuth state
DELETE FROM oauth_states WHERE created_at < now() - interval '10 minutes';
```

## 常见问题

### Q: OAuth 回调失败？

检查 `.env` 中的 `BASE_URL` 是否与 Google Cloud Console 中配置的授权重定向 URI 一致。回调地址格式为 `{BASE_URL}/api/oauth/callback`。

### Q: 如何刷新过期的 YouTube Token？

在频道列表点击「刷新Token」按钮，系统使用 refresh_token 自动获取新 Token。

### Q: 日志在哪里？

- **实时**：前端通过 API 轮询 `/api/tasks/{task_id}/logs/recent` 获取最新日志
- **持久化**：PostgreSQL `run_task_logs` 表
- **容器日志**：`docker-compose logs web`

### Q: 如何停止运行中的任务？

在任务管理或日志页面点击「停止」按钮。系统在 `run_tasks` 表设置 `stop_requested = true`，Pipeline 在下一个检查点检测到标志后优雅退出。

### Q: DeepFilter 下载失败？

`git-server-deploy.sh` 会在部署时预下载 DeepFilter 到 `data/deepfilter/`。如果网络不稳定导致下载失败，容器启动时 `entrypoint.sh` 会自动重试。

### Q: BGM 音乐库为空？

`git-server-deploy.sh` 会预下载 BGM 音乐包到 `data/music/`。如果下载失败，混音阶段会跳过 BGM。可手动下载：
```bash
wget "https://huggingface.co/datasets/oooooo1323/cm/resolve/main/Parisian%20Breeze.zip" -O /tmp/bgm.zip
unzip /tmp/bgm.zip -d data/music/
```

### Q: 如何重建镜像？

```bash
docker-compose build --no-cache web
docker-compose up -d
```

### Q: 如何完全重置？

```bash
# 停止并删除所有容器和数据卷
docker-compose down -v

# 重新启动
bash scripts/git-server-deploy.sh
```

> **警告**：`docker-compose down -v` 会删除 PostgreSQL 数据，请先备份！参考 `数据库备份恢复迁移指南.md`。
