# 部署指南

## 架构概览

| 组件 | 技术 | 说明 |
|------|------|------|
| Web 服务 | FastAPI + Uvicorn | 端口 59386，Jinja2 服务端渲染 |
| 数据库 | PostgreSQL 16 | Docker 内置，端口 5432 |
| 任务执行 | Python `threading.Thread` | 无需 Redis/Celery |
| 容器 | Docker Compose | `web` + `postgres` 两个服务 |

```
用户浏览器 → :59386 → FastAPI (web 容器)
                        ├── Jinja2 页面渲染
                        ├── API 路由 (频道/任务/书籍/配置/测试)
                        ├── 后台线程 (Pipeline 执行)
                        └── PostgreSQL (postgres 容器)
                              ├── books / audiobook_chapters
                              ├── channels / channel_configs
                              ├── run_tasks / run_task_logs
                              ├── global_settings / oauth_states
                              └── hf_jobs / hf_worker_stats
```

## 首次部署

### 前置条件

- VPS 或本地机器已安装 Docker + Docker Compose
- 已有 Git 仓库（GitHub/Gitea 等）

### 步骤 1：服务器克隆仓库

```bash
git clone https://github.com/YOUR_USER/YOUR_REPO.git /root/audiobook
cd /root/audiobook
```

### 步骤 2：配置环境变量

```bash
cp .env.example .env
nano .env
```

需要修改的配置：

| 变量 | 说明 | 示例 |
|------|------|------|
| `POSTGRES_PASSWORD` | 数据库密码（首次启动创建，之后不可改） | `my_strong_password_123` |
| `SECRET_KEY` | 加密密钥（64 位随机字符串） | `openssl rand -hex 32` 生成 |
| `BASE_URL` | 外部访问地址（OAuth 回调用） | `http://123.45.67.89:59386` |
| `APP_PASSWORD` | Web 界面登录密码（可选，默认 `inriynisse`） | `mypassword` |

> **重要**：`BASE_URL` 必须与 Google Cloud Console 中 OAuth 重定向 URI 一致。
> 回调地址格式为 `{BASE_URL}/api/oauth/callback`。

### 步骤 3：一键部署

```bash
bash scripts/git-server-deploy.sh
```

脚本自动完成：
1. `git pull` 拉取最新代码（脚本自身更新会自动重新执行）
2. 检查 `.env` 配置
3. 预下载 DeepFilter 二进制 + BGM 音乐库（持久化到宿主机 `data/` 目录）
4. 智能构建 Docker 镜像（仅在依赖/源码变更时重建）
5. 启动 `web` + `postgres` 容器
6. 健康检查（等待服务就绪）

首次部署约需 5-10 分钟（下载镜像 + 构建依赖 + 下载 DeepFilter/BGM）。
后续日常更新通常只需 10-30 秒（跳过构建，仅重启）。

### 步骤 4：验证

```bash
# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f web

# 浏览器访问
http://YOUR_SERVER_IP:59386
```

首次访问需输入登录密码（默认 `inriynisse`，可在 `.env` 中修改 `APP_PASSWORD`）。

API 文档：`http://YOUR_SERVER_IP:59386/api/docs`

## 开发模式

本地开发使用热重载模式，修改 `backend/` 下的 Python 代码后自动重载：

```bash
# 确保 .env 已配置
docker-compose -f docker-compose.yml -f docker-compose.dev.yml up
```

开发模式特点：
- `backend/` 目录挂载到容器，代码修改即时生效
- PostgreSQL 端口 5432 暴露到本地，方便用 GUI 工具调试
- Uvicorn `--reload` 自动检测文件变更

## Google OAuth 配置

### 1. 创建 Google Cloud 项目

1. 访问 [Google Cloud Console](https://console.cloud.google.com/)
2. 创建新项目（如 `audiobook-yt`）
3. 启用 YouTube Data API v3

### 2. 配置 OAuth 凭据

1. 进入「APIs & Services」→「Credentials」
2. 创建 OAuth 2.0 Client ID（Web application）
3. 添加授权重定向 URI：
   ```
   http://YOUR_SERVER_IP:59386/api/oauth/callback
   ```
   或使用域名：
   ```
   https://your-domain.com/api/oauth/callback
   ```
4. 下载 `client_secret.json`

### 3. 在系统中添加频道

1. 登录管理系统 → 频道管理 → 添加频道
2. 填写频道名和显示名
3. 上传下载的 `client_secret.json`
4. 点击「授权」→ 跳转 Google 完成授权
5. 授权成功后自动回调

> 如果 `BASE_URL` 变更，需要同步更新 Google Cloud Console 中的重定向 URI。

## 数据库管理

### 连接数据库

```bash
# 进入 PostgreSQL 容器
docker exec -it audiobook_postgres psql -U audiobook_app -d audiobook

# 从宿主机连接（需暴露端口）
psql -h 127.0.0.1 -U audiobook_app -d audiobook
```

### 数据库表说明

| 表名 | 说明 |
|------|------|
| `books` | 书籍库（含 book_data JSON、book_status 章节状态） |
| `audiobook_chapters` | TG 缓存章节（含 telegram_file_id、upload_status） |
| `channels` | 频道注册表 |
| `channel_configs` | 频道运行配置（95+ 参数 JSON） |
| `channel_runtime_settings` | 频道级运行时键值对 |
| `youtube_credentials` | YouTube OAuth Token |
| `modelscope_tokens` | AI 生图 Token |
| `book_processing_states` | 断点续跑状态 |
| `task_queue` | 原始任务队列 |
| `run_tasks` | Web 管理层任务记录（含 stop_requested） |
| `run_task_logs` | 任务日志持久化 |
| `oauth_states` | OAuth State 临时存储 |
| `global_settings` | 全局共享设置 |
| `hf_jobs` | HF 外包任务队列 |
| `hf_worker_stats` | HF Worker 统计 |

### 常用 SQL

```sql
-- 查看书籍总数
SELECT COUNT(*) FROM books;

-- 查看任务状态分布
SELECT status, COUNT(*) FROM run_tasks GROUP BY status;

-- 查看频道 OAuth 状态
SELECT channel_name, oauth_status FROM channels;

-- 重置卡住的 HF 任务
UPDATE hf_jobs SET status = 'pending', worker_id = NULL, claimed_at = NULL
  WHERE status = 'processing' AND claimed_at < now() - interval '24 hours';

-- 清理过期 OAuth state
DELETE FROM oauth_states WHERE created_at < now() - interval '10 minutes';
```

## 常见问题

### Q: OAuth 回调失败？

检查 `.env` 中的 `BASE_URL` 是否与 Google Cloud Console 中配置的授权重定向 URI 一致。回调地址格式为 `{BASE_URL}/api/oauth/callback`。

### Q: 如何刷新过期的 YouTube Token？

在频道列表点击「刷新Token」按钮，系统使用 refresh_token 自动获取新 Token。

### Q: 日志在哪里？

- **实时**：前端通过 API 轮询 `/api/tasks/{task_id}/logs/recent` 获取最新日志
- **持久化**：PostgreSQL `run_task_logs` 表
- **容器日志**：`docker-compose logs web`

### Q: 如何停止运行中的任务？

在任务管理或日志页面点击「停止」按钮。系统在 `run_tasks` 表设置 `stop_requested = true`，Pipeline 在下一个检查点检测到标志后优雅退出。

### Q: DeepFilter 下载失败？

`git-server-deploy.sh` 会在部署时预下载 DeepFilter 到 `data/deepfilter/`。如果网络不稳定导致下载失败，容器启动时 `entrypoint.sh` 会自动重试。

### Q: BGM 音乐库为空？

`git-server-deploy.sh` 会预下载 BGM 音乐包到 `data/music/`。如果下载失败，混音阶段会跳过 BGM。可手动下载：
```bash
wget "https://huggingface.co/datasets/oooooo1323/cm/resolve/main/Parisian%20Breeze.zip" -O /tmp/bgm.zip
unzip /tmp/bgm.zip -d data/music/
```

### Q: 如何重建镜像？

```bash
docker-compose build --no-cache web
docker-compose up -d
```

### Q: 如何完全重置？

```bash
# 停止并删除所有容器和数据卷
docker-compose down -v

# 重新启动
bash scripts/git-server-deploy.sh
```

> **警告**：`docker-compose down -v` 会删除 PostgreSQL 数据，请先备份！参考 `数据库备份恢复迁移指南.md`。
